# cpp-algorithms

一个用于记录 C++ 算法题练习、题解思路和备赛笔记的仓库, 当前主要包含蓝桥杯第十六届 B 组相关题目代码.

## 项目内容

- `第十六届B组/`.
  - 按比赛目录组织的题目内容.
  - 题目逐步统一为一题一个文件夹.
- `_template_problem/`.
  - 标准题目模板目录.
  - 新题可参考其中的 `Solution.md`, `Problem.md`, `Math.md` 和示例 `.cpp`.
- `备赛经验和技巧.md`.
  - 备赛过程中整理的常用技巧、输入输出优化、调试经验等.
- `CLAUDE.md`.
  - 仓库级协作规范.
  - 说明题目模板、Markdown 链接规则和 AI 协作约定.
- `helloworld.cpp`.
  - 本地环境初始化时的示例文件, 可忽略其算法意义.
- `编译文件夹/`.
  - 本地编译产生的可执行文件目录, 不纳入版本管理.

## 题目组织方式

- 优先按比赛或专题分目录管理.
- 每道题尽量使用一个独立文件夹.
- `Solution.md` 作为单题主入口, 负责链接 `Problem.md`, `Math.md` 和推荐代码.
- 文档之间统一使用 Markdown 相对路径链接, 便于在 VSCode、WSL、Obsidian 和 GitHub 中同步使用.
- 所有正文统一使用半角英文标点, 句号统一使用 `.`.
- 不使用中文符号, 包括 `。` 和 `、`.
- 新题可直接参考 [`_template_problem/Solution.md`](./_template_problem/Solution.md).

## 代码风格

仓库中的代码以竞赛/刷题风格为主, 特点包括.

- 优先追求题目通过与实现简洁.
- 常见写法如 `ios::sync_with_stdio(false);`, `cin.tie(0);`.
- 在需要时使用 `long long` 避免溢出.
- 同一题可能保留多个版本, 便于比较不同思路.

## 运行方式

### 使用 g++ 编译

```bash
g++ -std=c++17 -O2 -o main "第十六届B组/03可分解的正整数.cpp"
./main
```

如需运行其他题目, 将文件路径替换为对应的 `.cpp` 文件即可.
